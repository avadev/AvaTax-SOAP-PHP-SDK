method cancelTax [line 186]

a cancelTax( cancelTaxRequest &$cancelTaxRequest)

Cancels a previously calculated tax; This is for use as a compensating action when posting on the client fails to complete.

This is only available for saved tax document types (Sales Invoices, Purchase Invoices). A document that is saved but not posted will be deleted if canceled. A document that has been posted will revert to a saved state if canceled (in this case CancelTax should be called with a getCancelCode of PostFailed). A document that has been committed cannot be reverted to a posted state or deleted. In the case that a document on the client side no longer exists, a committed document can be virtually removed by calling CancelTax with a CancelCode of DocDeleted. The record will be retained in history but removed from all reports.

A document can be indicated solely by the getDocId if it is known. Otherwise the request must specify all of getCompanyCode, getDocCode, and getDocType in order to uniquely identify the document.

method reconcileTaxHistory [line 222]

A reconcileTaxHistory( reconcileTaxHistoryRequest &$reconcileTaxHistoryRequest)

Reconciles tax history to ensure the client data matches the AvaTax history.

The Reconcile operation allows reconciliation of the AvaTax history with the client accounting system. It must be used periodically according to your service contract.

Because there may be a large number of documents to reconcile, it is designed to be called repetitively until all documents have been reconciled. It should be called until no more documents are returned. Each subsequent call should pass the previous results getLastDocId.

When all results have been reconciled, Reconcile should be called once more with getLastDocId equal to the last document code processed and isReconciled set to true to indicate that all items have been reconciled. If desired, this may be done incrementally with each result set. Just send Reconciled as true when requesting the next result set and the prior results will be marked as reconciled.

The #postTax, #commitTax, and #cancelTax operations can be used to correct any differences. #getTax should be called if any committed documents are out of balance (getTotalAmount or getTotalTax don't match the accounting system records). This is to make sure the correct tax is reported.
